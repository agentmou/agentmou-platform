# Documentation Archive

This directory stores material that is useful for historical context but is not
part of the primary newcomer or operator reading path.

## Contents

- `context/` keeps the original repo-wide context document that shaped the
  project direction.
- `architecture/` keeps refactor logs and dated audits that describe how the
  repository changed over time.
- `planning/` keeps archived execution plans, epic templates, and retired
  planning artifacts.

## How To Use The Archive

- Use the archive to understand why the repo changed or to recover historical
  decisions that are no longer part of the active docs surface.
- Do not treat archived documents as the operational source of truth.
- Prefer the live documents under [`docs/`](../README.md) when you need the
  current repository shape, commands, or runbooks.
